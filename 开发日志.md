# 开发日志

## 2026.1.12
问题：
    当前数据采集系统出现了一些问题，当外部输入源是正弦波时，界面的图表显示的不是正弦波，竟然是一段有规律的混乱波形，当取消外部输入源时，界面的图表显示的依然是之前的规律混乱波形。

原因：
    可以确定问题是采集卡sdk的问题，由于iResult = XCDotNETAPI.usb1602API.AD_GetBulkData(mHandle, buffer, BufferSize, ref nBytes);中iResult =fasle，nBytes=0。而数据采集线程的仍在 for (i = 0; i < block.Buffer.Length / 2; i++) 中循环处理数据，且复用的数组池数组中仍有旧数据。这就导致每一轮循环中既有新数据又有旧数据，从而导致规律混乱波形。
解决方案：
    直接调用厂商硬件的原生 C API
