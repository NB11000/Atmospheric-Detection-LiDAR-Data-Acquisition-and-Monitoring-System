# 波形重连回调并发刷屏

**日期**: 2026-07-05
**状态**: 已修复
**文件**: `WebAPI/Service/MqttRpcBackgroundService.cs` → `OnWaveformDisconnectedAsync`

---

## 现象

波形 MQTT 连接断开后，日志中每隔 3 秒出现多条相同的 "波形连接重连失败" Warning，形成刷屏。

## 根因

对比 `OnDisconnectedAsync`（主连接）和 `OnWaveformDisconnectedAsync`（波形连接）两个断连回调：

| 维度 | 主连接 | 波形连接（修复前） |
|---|---|---|
| 重入保护 | `_reconnectLock.WaitAsync(0)` 非阻塞抢锁 | **无** |
| 退避策略 | 1s → 2s → 4s → ... → 60s 上限 | 恒定 3s |

### 具体原因

1. **无重入保护导致并发 while 循环**：MQTTnet 在 `ConnectAsync` 调用失败后也会触发 `DisconnectedAsync` 事件。当第一个断连回调进入 `while` 循环并尝试重连失败后，该事件再次触发，第二个回调也进入 `while` 循环，如此堆叠 N 个并发循环同时打印日志。
2. **无指数退避**：即使只有单个循环，恒定 3 秒意味着长时间断连（如 Broker 宕机数小时）会产生大量重复日志。

## 修复

参照 `OnDisconnectedAsync` 的实现模式：

1. 新增 `_waveformReconnectLock`（`SemaphoreSlim(1, 1)`）提供波形重连的重入保护
2. `OnWaveformDisconnectedAsync` 入口处用 `WaitAsync(0)` 非阻塞抢锁，抢不到则直接 `return`
3. 退避从 1s 开始指数翻倍，上限复用 `ReconnectDelaySeconds` 配置（默认 60s）
4. `StopAsync` 中清理波形连接前先等待重连锁释放，避免竞态

### 变更摘要

```diff
+ private readonly SemaphoreSlim _waveformReconnectLock = new(1, 1);

  private async Task OnWaveformDisconnectedAsync(args)
  {
-     var delay = TimeSpan.FromSeconds(3);
+     if (!await _waveformReconnectLock.WaitAsync(0))
+     {
+         _logger.LogDebug("波形重连已在进行中，跳过重复触发");
+         return;
+     }
+     try { ... } finally { _waveformReconnectLock.Release(); }
+
+     var delay = TimeSpan.FromSeconds(1);
+     var cap = TimeSpan.FromSeconds(settings.ReconnectDelaySeconds > 0 ? ... : 60);
+     delay = TimeSpan.FromSeconds(Math.Min(delay.TotalSeconds * 2, cap.TotalSeconds));
  }
```
