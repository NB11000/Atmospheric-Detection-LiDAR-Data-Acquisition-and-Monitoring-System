# DetectionPublisherService — 检测告警 MQTT 发布服务

## Context

ConsoleApp1 子进程的 Detection 线程已通过 gRPC 将 `SIGNAL_OBSTRUCTION` 告警上报到 WebAPI，但 WebAPI 没有专门的服务接收结构化检测数据并发布到专属 MQTT topic。当前检测数据作为纯文本 Error 消息混杂在 `daq/{id}/events/state_changed` 中，StructuredSample 上下文（Timestamp、CH1、CH2、Time）全部丢失。

目标：新增 `DetectionPublisherService`，和现有 3 个 `IAcquisitionBoundService`（WaveformPublishService、PersistenceService、LowFrequencyPublisher）保持一致模式，将结构化检测告警发布到 `daq/{id}/detection/alerts`。

## Design Decisions (from /grill-with-docs)

| 决策 | 选择 |
|---|---|
| 数据传递 | 扩展 proto，`AdResponse.data` 字段（google.protobuf.Any）打包 `DetectionAlert` |
| MQTT topic | `daq/{machineId}/detection/alerts`，QoS 1，不 Retain |
| 生命周期 | 实现 `IAcquisitionBoundService`，`RequiresMqttConnection = true` |
| 告警分级 | 当前仅 `SIGNAL_OBSTRUCTION`，severity=warning，预留扩展点 |
| 传输粒度 | 逐条发送（一个检测结果 → 一条 gRPC → 一条 MQTT） |
| 内部事件 | GrpcServiceImpl 直接注入 DetectionPublisherService 调用 |
| Payload | 精简格式：Timestamp, Time, CH1, CH2, alarmType, severity, utcTime |

## Implementation Plan

**Files:** 3 处 proto 文件（内容相同）

新增消息定义（在 `AdResponse` 之后）：

```protobuf
message DetectionAlert {
  string alarm_type = 1;    // e.g. "SIGNAL_OBSTRUCTION"
  string severity = 2;      // "info" | "warning" | "critical"
  int64 timestamp = 3;      // globalSampleIndex (采样点序号)
  int64 time_ticks = 4;     // Stopwatch.GetTimestamp() ticks
  double ch1 = 5;           // 通道1电压
  double ch2 = 6;           // 通道2电压
}
```

### Step 2: ConsoleApp1/GrpcClient — 新增 `SendDetectionAlert()`

**File:** `ConsoleApp1/Service/GrpcClient.cs`

新增方法，与 `SendErrorMessage` 并列：

```csharp
public static void SendDetectionAlert(string alarmType, string severity,
    long timestamp, long timeTicks, double ch1, double ch2)
{
    var alert = new DetectionAlert
    {
        AlarmType = alarmType,
        Severity = severity,
        Timestamp = timestamp,
        TimeTicks = timeTicks,
        Ch1 = ch1,
        Ch2 = ch2
    };

    var response = ResponsePool.Get();
    response.ResponseId = Guid.NewGuid().ToString("N");
    response.MessageType = "Detection";   // 新 message_type
    response.ErrorCode = "NONE";
    response.ProcessId = _processId;
    response.Data = Any.Pack(alert);      // 打包结构化数据

    ResponseQueue.Enqueue(response);
}
```

新增 using: `using Google.Protobuf.WellKnownTypes;`（已存在，第 5 行）

### Step 4: WebAPI — 新建 `DetectionPublisherService`

**File:** `WebAPI/Service/DetectionPublisherService.cs`（新建）

参考 `LowFrequencyPublisher.cs` 的结构：
- `IAcquisitionBoundService` 实现（`RequiresMqttConnection = true`）
- `lock` + `_isRunning` 双重检查保证 `Start()`/`Stop()` 幂等
- 内部维护 `ConcurrentQueue<DetectionAlert>` 缓冲池
- 后台 `Task` 循环消费队列 → 调用 `MqttEventPublisher` 发布到 `daq/{id}/detection/alerts`
- 核心方法 `OnAlertReceived(DetectionAlert alert)` 由 `GrpcServiceImpl` 调用

```csharp
public class DetectionPublisherService : IAcquisitionBoundService, IDisposable
{
    private readonly MqttEventPublisher _mqttPublisher;
    private readonly IOptionsMonitor<MqttSettings> _mqttSettings;
    private readonly TimeHelper _timeHelper;  // 或直接内联计算 UTC
    private readonly ILogger<DetectionPublisherService> _logger;

    private readonly ConcurrentQueue<DetectionAlert> _alertQueue = new();
    private readonly AutoResetEvent _alertSignal = new(false);
    private CancellationTokenSource? _cts;
    private Task? _loopTask;
    private readonly object _lock = new();
    private bool _isRunning;

    public bool RequiresMqttConnection => true;

    // Start/Stop by AcquisitionLifecycleCoordinator
    public void Start() { ... }
    public void Stop() { ... }

    // Called by GrpcServiceImpl
    public void OnAlertReceived(DetectionAlert alert)
    {
        _alertQueue.Enqueue(alert);
        _alertSignal.Set();
    }
}
```

Payload JSON:
```json
{
  "alarmType": "SIGNAL_OBSTRUCTION",
  "severity": "warning",
  "timestamp": 12345678,
  "time": 9876543210,
  "ch1": 0.0003,
  "ch2": 2.4510,
  "utcTime": "2026-05-06T10:30:00.123Z"
}
```

MQTT 主题: `daq/{machineId}/detection/alerts`
QoS: AtLeastOnce，Retain: 否
