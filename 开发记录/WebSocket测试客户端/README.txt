WebSocket测试客户端 - 使用说明
====================================

## 概述
此测试客户端用于验证数据采集系统的WebSocket数据流功能。它会连接到WebAPI服务器的WebSocket端点，接收实时数据帧，并显示统计信息。

## 前提条件
1. .NET 8 SDK 已安装
2. WebAPI服务器正在运行 (监听5135端口)
3. 数据采集系统已启动并开始采集

## 快速开始

### 方法1：使用脚本编译并运行
在 `WebSocket测试客户端` 目录中打开命令行，执行：

```bash
dotnet run
```

### 方法2：手动编译运行
```bash
# 恢复依赖
dotnet restore

# 编译项目
dotnet build

# 运行程序
dotnet run
```

### 方法3：指定服务器地址
```bash
dotnet run -- ws://192.168.1.100:5135/ws/ui-data
```

## 故障排除

### 1. 连接被拒绝
```
❌ WebSocket错误: 无法连接到远程服务器
```
解决方法：
- 确认WebAPI服务器正在运行 (`dotnet run` 在 WebAPI 目录)
- 确认防火墙允许5135端口
- 确认服务器地址正确

### 2. 无数据接收
```
✅ 连接成功！
📊 开始接收数据帧...
```
但无后续输出。

解决方法：
- 确认数据采集已开始 (点击"开始采集"按钮)
- 确认子进程(ConsoleApp1)正常运行
- 检查WebAPI服务器日志

### 3. 帧大小不匹配
```
⚠️  警告: 收到异常帧大小 12000字节 (预期16,000字节)
```
解决方法：
- 确认服务器和客户端使用相同的数据格式
- 检查WebAPI的 `HandleWebSocketConnection` 方法

## 技术细节
- **连接地址**: `ws://localhost:5135/ws/ui-data`
- **数据格式**: 16,000字节二进制帧
  - 前8,000字节: 通道1的1000个double (64位浮点数)
  - 后8,000字节: 通道2的1000个double
- **帧率**: 约30 FPS (每33毫秒一帧)
- **带宽**: 约480 KB/s (3.84 Mbps)

## 注意事项
1. 测试前确保WebAPI服务器已启动
2. 需要先开始数据采集才有数据流
3. 按Ctrl+C可安全停止测试
4. 支持自定义服务器地址参数
