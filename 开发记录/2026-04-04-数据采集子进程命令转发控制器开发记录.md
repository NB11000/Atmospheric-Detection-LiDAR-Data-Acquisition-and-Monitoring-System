## 开发背景

基于用户需求，需要创建一个专用的ASP.NET Core控制器，用于转发前端HTTP请求到数据采集子进程。该控制器需要：
1. 基于ConsoleApp1项目中的GrpcClient.cs功能
2. 提供HTTP API接口供前端调用
3. 通过gRPC服务向数据采集子进程发送指令
4. 简化API设计，移除不必要的复杂性

## 设计决策

### 1. 控制器专用化
- 将通用的ClientController修改为专用的命令转发控制器
- 移除多客户端支持，专注于单一数据采集子进程
- 控制器名称保持`ClientController`，但功能专一化

### 2. 客户端ID硬编码
- 根据Program.cs中的使用模式，客户端ID硬编码为`"数据采集子进程"`
- 避免在URL中传递参数，简化前端调用
- 与现有系统保持一致（Program.cs第181行使用相同ID）

### 3. 简化请求模型
- 取消`SendCommandRequest`类（仅包含单个Command属性）
- 直接使用`[FromBody] string command`接收指令
- 减少嵌套，简化API调用

### 4. 路由设计
- 路由从`api/[controller]`改为`api/collector/command`
- 清晰表达控制器用途：数据采集子进程命令转发
- 便于前端理解和调用

## 实现的API端点

### 主要端点

| 方法 | 路径 | 描述 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | `/api/collector/command/status` | 检查数据采集子进程连接状态 | 无 | `{ ClientId, Connected, Timestamp }` |
| POST | `/api/collector/command` | 发送指令并等待响应 | `"OPEN_DEVICE"`（字符串） | `AdResponse`对象 |
| POST | `/api/collector/command/async` | 异步发送指令（不等待响应） | `"START_AD"`（字符串） | 202 Accepted |

### 快捷指令端点

| 方法 | 路径 | 描述 | 对应指令 |
|------|------|------|----------|
| POST | `/api/collector/command/open` | 打开采集卡设备 | `OPEN_DEVICE` |
| POST | `/api/collector/command/open-again` | 重新打开采集卡设备 | `OPEN_DEVICE_AGAIN` |
| POST | `/api/collector/command/close` | 关闭采集卡设备 | `CLOSE_DEVICE` |
| POST | `/api/collector/command/start` | 开始数据采集 | `START_AD` |
| POST | `/api/collector/command/stop` | 停止数据采集 | `STOP_AD` |
| POST | `/api/collector/command/ping` | 心跳检测 | `PING` |
| POST | `/api/collector/command/exit` | 优雅退出 | `EXIT` |
| POST | `/api/collector/command/config` | 读取配置 | `CONFIG` |

## 支持的gRPC指令

控制器支持ConsoleApp1项目中GrpcClient.cs定义的所有指令：

1. **设备控制指令**
   - `OPEN_DEVICE`: 打开采集卡设备
   - `OPEN_DEVICE_AGAIN`: 重新打开采集卡设备
   - `CLOSE_DEVICE`: 关闭采集卡设备

2. **采集控制指令**
   - `START_AD`: 开始数据采集
   - `STOP_AD`: 停止数据采集

3. **系统管理指令**
   - `PING`: 心跳检测
   - `EXIT`: 优雅退出客户端进程
   - `CONFIG`: 读取配置文件

## 错误处理

控制器实现了完整的错误处理机制：

1. **客户端未连接**: 返回404 Not Found
2. **指令超时**: 返回504 Gateway Timeout（gRPC服务默认10秒超时）
3. **指令为空**: 返回400 Bad Request
4. **服务器异常**: 返回500 Internal Server Error
5. **gRPC异常**: 捕获并转换为适当的HTTP状态码

## 代码示例

### 控制器核心方法
```csharp
[HttpPost]
public async Task<IActionResult> SendCommand([FromBody] string command)
{
    if (string.IsNullOrEmpty(command))
    {
        return BadRequest("指令不能为空");
    }

    try
    {
        _logger.LogInformation($"向数据采集子进程发送指令：{command}");
        var response = await _grpcService.SendCommandToClientAndWaitResponse(ClientId, command);
        return Ok(response);
    }
    catch (Exception ex) when (ex.Message.Contains("未连接"))
    {
        _logger.LogWarning($"数据采集子进程未连接");
        return NotFound($"数据采集子进程未连接");
    }
    catch (TimeoutException ex)
    {
        _logger.LogWarning($"等待数据采集子进程响应超时");
        return StatusCode(504, $"等待客户端响应超时：{ex.Message}");
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, $"向数据采集子进程发送指令[{command}]时发生异常");
        return StatusCode(500, $"内部服务器错误：{ex.Message}");
    }
}
```
