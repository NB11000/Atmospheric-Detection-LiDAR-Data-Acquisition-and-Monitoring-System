# 数据采集与检测系统 V2.0 数据流分析

---

## 一、系统架构总览

项目由三个独立进程构成：

| 进程 | 项目 | 角色 |
|------|------|------|
| **主进程** | `WebAPI` (ASP.NET Core) | gRPC Server + MQTT Client + HTTP API + SignalR Hub + WebSocket Server + 共享内存创建者 |
| **子进程** | `ConsoleApp1` (.NET 8 Console) | gRPC Client + 数据采集驱动 + 共享内存写入者 |
| **桌面客户端** | `AvaloniaApplication1` | UI 展示层（MQTT 订阅者 + WebSocket 消费者） |

**进程间通信**：gRPC 双向流  
**数据共享**：内存映射文件 (MMF)  
**外部通信**：MQTT Broker (EMQX, TLS/SSL)

---

## 二、数据源 (Data Sources)

### 2.1 硬件数据源

| 数据源 | 接口 | 数据类型 | 生产者 |
|--------|------|----------|--------|
| **USB1602 数据采集卡** | USB 原生 DLL (`USB1602.dll`) | 原始模拟采样数据（16-bit ADC，每采样点 2 字节） | ConsoleApp1 子进程 |
| **CNI 激光器** | RS-232 串口 | 二进制控制协议（0x55 0xAA 帧头 + 校验和） | 无主动数据上报，仅响应指令 |

### 2.2 配置数据源

| 配置项 | 文件 | 路径 | 加载方 |
|--------|------|------|--------|
| 采集卡配置 | `DeviceConfig.json` | 主进程 `Config/` | 主进程 → 通过命令行参数传递路径给子进程 |
| 激光雷达配置 | `RadarConfig.json` | 主进程 `Config/` | 仅主进程读取 |
| MQTT 连接配置 | `appsettings.json` | WebAPI 根目录 | 主进程启动时读取 |

## 三、数据处理流 (Processing Pipeline)

### 3.1 数据采集流（硬件 → 共享内存 → MQTT/WebSocket）

这是系统的**核心数据流**，分为 5 个阶段：

```
+-----------------------------------------------------------------------------+
| 阶段1: 硬件读取 (ConsoleApp1子进程, AD_Controlcs.ADWork线程)                  |
|                                                                             |
|  USB1602 采集卡                                                              |
|      | USB 批量读取 (184,320 字节/帧)                                        |
|      v                                                                       |
|  byte[] buffer (ArrayPool租用, ~200KB)                                       |
|      |                                                                       |
|      +--> Channel<Data_Block> (有界通道, 容量64, 满时丢弃最旧)                  |
+-----------------------------------------------------------------------------+
                                    |
                                    v
+-----------------------------------------------------------------------------+
| 阶段2: 数据解码与校准 (ADDraw线程)                                            |
|                                                                             |
|  Channel<Data_Block> 读取                                                    |
|      |                                                                       |
|      | 根据 chSel (1=CH1, 2=CH2, 3=双通道) 解析数据布局                       |
|      |                                                                       |
|      +-- 双通道(CH3): 每4字节包含 CH1_sample + CH2_sample (各2字节交错)        |
|      +-- 单通道: 每2字节一个采样点                                            |
|      |                                                                       |
|      | 转换公式 (每个采样点):                                                  |
|      |   dfRet = buffer[i] + buffer[i+1] * 256  (16位小端)                    |
|      |   dfRet = dfRet & 0x0000ffff         (掩码保护)                        |
|      |   Fun_DataToV(ref dfRet, Gain)       (量程/增益转换)                   |
|      |   dfRet = k * dfRet + b              (线性校准, k/b从硬件获取)          |
|      |   dfRet = Math.Round(dfRet, 5)       (精度截断)                       |
|      |                                                                       |
|      +--> Channel<Voltage_block> (分析通道, 容量64)                            |
|      +--> Channel<UI_Display>   (UI通道, 容量10)                              |
+-----------------------------------------------------------------------------+
                                    |
                                    v
+-----------------------------------------------------------------------------+
| 阶段3: 降采样与共享内存写入 (UI线程)                                           |
|                                                                             |
|  Channel<UI_Display> 读取 (同步阻塞, 每批次约92,160采样点)                     |
|      |                                                                       |
|      | Span<double> 拷贝前1000点到 text1/text2 (栈上数组, 零分配)              |
|      |                                                                       |
|      v                                                                       |
|  uISharedBuffer.WriteSampleBatch(text1, text2, 1000)                        |
|      |                                                                       |
|      | 写入 MMF "DAQ_UI_RING_BUFFER" (环形缓冲区, 容量30,000点x2通道)          |
|      | Interlocked.Add 原子递增 WriteIndex                                  |
|      |                                                                       |
|      v                                                                       |
|  共享内存 MMF (跨进程可见)                                                    |
+-----------------------------------------------------------------------------+
           |                              |
           v (主进程读取)                  v (主进程读取)
+-----------------------------+  +-------------------------------------------+
| 阶段4A: WebSocket输出        |  | 阶段4B: MQTT 波形输出                      |
| (Tool.cs)                   |  | (WaveformPublishService)                  |
|                             |  |                                           |
| 每33ms触发一次               |  | 由 AcquiringStateChanged 事件驱动           |
| ReadLatestFrame()           |  | PeriodicTimer(默认100ms间隔)               |
| unsafe指针拷贝               |  | ReadLatestFrame() -> double[]               |
| -> WebSocket Binary帧       |  | Buffer.BlockCopy -> byte[]                  |
|   16KB/帧                   |  | PublishWaveformDataAsync()                 |
| -> /ws/ui-data 端点         |  | -> MQTT daq/{id}/waveform/ch1 (QoS0)       |
|                             |  | -> MQTT daq/{id}/waveform/ch2 (QoS0)       |
+-----------------------------+  +-------------------------------------------+
```

**关键文件位置**：

- 阶段1: `ConsoleApp1/Service/AD_Controlcs.cs:348` (`ADWork`)
- 阶段2: `ConsoleApp1/Service/AD_Controlcs.cs:450` (`ADDraw`)
- 阶段3: `ConsoleApp1/Service/AD_Controlcs.cs:777` (`UI`)
- 阶段4A: `WebAPI/Tools/Tool.cs:75` (`HandleWebSocketConnection`)
- 阶段4B: `WebAPI/Service/WaveformPublishService.cs:164` (`RunLoopAsync`)

```
|  |  system-state                        |                                 |

|  | LogHandler (5个方法):                |                                 |
|  |  logs-query / -by-level / -stats     |                                 |
|  |  logs-clear / -health                |                                 |
|  +--------------------------------------+                                 |
+-----------------------------------+---------------------------------------+
                                    |
                                    v
+---------------------------------------------------------------------------+
|  GrpcServiceImpl (主进程)                                                   |
|                                                                           |
|  SendCommandToClientAndWaitResponse("数据采集子进程", command)              |
|      |                                                                    |
|      | 生成 requestId (GUID)                                              |
|      | 创建 TaskCompletionSource<AdResponse> (同步等待器)                   |
|      | 写入 _commandResponseWaiters 字典                                    |

|      v                                                                    |
|  gRPC HTTP/2 双向流 (AdRequest / AdResponse)                              |
+-----------------------------------+---------------------------------------+
                                    |
                                    v
+---------------------------------------------------------------------------+
|  GrpcClient (ConsoleApp1 子进程)                                            |
|                                                                           |
|  Communicate() 双向流                                                      |
|  HandleServerCommand(AdRequest)                                           |
|      |                                                                    |
|      | switch(request.Command):                                           |
|      +-- OPEN_DEVICE     -> AD_Controlcs.Device_Opened()                  |
|      +-- OPEN_DEVICE_AGAIN -> AD_Controlcs.Device_Opened_again()          |

|      +-- START_AD        -> AD_Controlcs.start() (启动4线程)                |
|      +-- STOP_AD         -> AD_Controlcs.stop()  (停止4线程)                |
|      +-- PING            -> 返回 "PONG"                                   |
|      +-- EXIT            -> 优雅退出流程                                    |
|      +-- CONFIG_UPDATE   -> ConfigHelper.ReadDeviceConfig()               |
|      +-- CONFIG_READ     -> ConfigHelper.ReadDeviceConfig()               |
|      +-- GET_COLLECTOR_STATE -> AD_Controlcs.GetRuntimeState()            |
|      |                                                                    |
|      v                                                                    |
|  AdResponse -> ResponseQueue (ConcurrentQueue) -> 集中发送线程              |
|  _responseStreamWriter.WriteAsync(response)                               |
+-----------------------------------+---------------------------------------+
                                    |
                                    v
+---------------------------------------------------------------------------+
|  硬件执行 (USB1602.dll / 串口)                                              |
|                                                                           |
|  Device_Opened:  USB1602.USB1602_OpenDevice(deviceId) -> mHandle          |
|  start():        设置通道/量程/频率/时钟/触发模式 -> 启动4线程                |
|  stop():         停止4线程 -> USB1602_ADStop -> USB1602_CLRRAM            |

+---------------------------------------------------------------------------+
```

### 3.3 系统状态流（状态推断 → 缓存 → 事件发布）

```
+-----------------------------------------------------------------------------+
|  状态变更触发源                                                              |
|                                                                             |
|  +-- gRPC 子进程连接/断开 --> collector_connected / collector_disconnected   |
|  +-- gRPC 命令响应         --> UpdateStateFromCommandResponse()              |
|  +-- gRPC 数据上报         --> UpdateStateFromDataReport()                   |
|  +-- gRPC 错误上报         --> UpdateStateFromError()                        |
|  +-- 激光器串口操作        --> CniLaser.UpdateLaserStateCache()              |
+------------------------------------+----------------------------------------+
                                     |
                                     v
+-----------------------------------------------------------------------------+
|  SystemStateService (单例, 状态机核心)                                         |
|                                                                             |
|  维护两个 volatile 缓存 (不可变对象替换模式, 线程安全):                          |
|                                                                             |
|  _cachedCollectorState: CollectorStateDto                                    |
|    +-- ProcessConnected (子进程是否连接)                                     |
|    +-- DeviceOpened     (采集卡是否打开)                                      |
|    +-- Acquiring        (是否正在采集)                                        |
|    +-- Handle           (设备句柄)                                           |
|    +-- LastMessage      (最近状态描述)                                        |
|                                                                             |
|  _cachedLaserStateDto: LaserStateDto                                         |
|    +-- SerialConnected  (串口是否连接)                                        |
|    +-- EmissionOn       (激光是否开启)                                        |
|    +-- PortName         (串口名称)                                           |
|    +-- LastMessage      (最近状态描述)                                        |
|                                                                             |
|  状态推断逻辑:                                                                |
|  +--------------------+------------------------------------+                 |
|  | 收到命令响应         | 推断规则                            |                 |
|  +--------------------+------------------------------------+                 |
|  | OPEN_DEVICE        | DeviceOpened = (MHandle > 0)      |                 |
|  | CLOSE_DEVICE       | DeviceOpened=false, Acquiring=false|                |
|  | START_AD           | Acquiring = (Content=="AD_STARTED")|                |
|  | STOP_AD            | Acquiring = false                  |                 |
|  | EXIT               | 全部重置                            |                 |
|  | DEVICE_DISCONNECTED| DeviceOpened=false, Acquiring=false|                |
|  | ACQUISITION_FAILED | Acquiring = false                  |                 |
|  | DEVICE_OPEN_FAILED | DeviceOpened = false               |                 |
|  +--------------------+------------------------------------+                 |
|                                                                             |
|  AcquiringStateChanged 事件 (状态驱动波形发布):                                |
|  Acquiring: false->true  -> 触发 WaveformPublishService.Start()              |
|  Acquiring: true->false  -> 触发 WaveformPublishService.Stop()               |
+------------------------------------+----------------------------------------+
                                     |
                                     v
+-----------------------------------------------------------------------------+
|  状态输出: 系统统一快照 (零 IPC, 全本地缓存读取)                                |
|                                                                             |
|  GetSystemState() -> SystemStateDto {                                        |
|    Server:  { IsApiAlive, Timestamp }                                       |
|    Collector: CollectorStateDto (从缓存)                                     |
|    Laser: LaserStateDto (从缓存)                                             |
|    UiHints: UiHintStateDto (根据硬件状态推导可操作按钮)                       |
|  }                                                                          |
|                                                                             |
|  MqttEventPublisher 发布到 MQTT (QoS 1):                                     |
|    -> daq/{machineId}/events/state_changed                                   |
|                                                                             |
|  SignalR Hub (兼容通道):                                                     |
|    -> /hubs/system-state (PublishStateChangedAsync)                          |
+-----------------------------------------------------------------------------+
```

## 四、存储机制 (Storage Mechanisms)

| 存储类型 | 实现 | 容量/限制 | 用途 | 生命周期 |
|----------|------|-----------|------|----------|
| **共享内存 (MMF)** | `UISharedBuffer` (`DAQ_UI_RING_BUFFER`) | 30,000点 x 2通道 x 8字节 ≈ 480KB | 双通道降采样波形数据 | 进程存活期间 |
| **共享内存 (MMF)** | `RawRingBuffer` (`DAQ_RAW_BUFFER`) | 可配置 | 原始采样数据（已定义但当前未活跃使用） | 进程存活期间 |
| **状态缓存** | `SystemStateService` volatile 字段 | 内存内, ~100字节 | 采集卡/激光器实时状态 | 应用程序生命周期 |
| **内存日志** | Serilog InMemory Sink | 内存内, 逐日滚动文件 | 运行时日志查询 | 应用程序生命周期 |
| **文件日志** | Serilog File Sink | 10MB/文件, 保留31天 | 持久化日志 | 按天滚动 |
| **配置文件** | `DeviceConfig.json` / `RadarConfig.json` | JSON 文件 | 采集卡参数、激光器参数 | 持久化 |
| **MQTT 保留消息** | Broker 端存储 (Retain Flag) | Broker 配置决定 | 设备报警最新状态 | 被新消息替换前 |

## 七、关键技术要点

| 组件 | 核心技术 | 说明 |
|------|----------|------|
| gRPC 双向流 | `stream AdResponse returns stream AdRequest` | 替代旧版 NetMQ，单长连接承载所有指令与数据上报 |
| 共享内存 | `MemoryMappedFile` + `unsafe` 指针 | 双区环形缓冲 (UISharedBuffer + RawRingBuffer)，无锁设计 |
| MQTT RPC | `$rpc/{machineId}/{method}/{corrId}` 主题模式 | 基于 MQTT 主题路由模拟 RPC 调用，4 类 Handler 共 26 个方法 |
