# 大气检测激光雷达数据解析链路 — 实施计划

> 修订日期：2026-05-01，依据《高频数据采集与分发系统软件架构说明文档 V1.3》同步修订，新增数据检测与工况判别模块。

## 1. 现状分析

### 已有架构
- **ConsoleApp1**（数据采集子进程）：4 个线程通过 Channel 传递数据
  - `ADWork` → `channel` → `ADDraw` → `Analysischannel` → `Analysis`（目标嵌入点）
  - `ADDraw` 已实现：原始 ADC 字节 → 模拟电压值（`AD_Controlcs.cs:569-625`）
  - `Analysis` 线程（`AD_Controlcs.cs:708-770`）当前为空占位 `// 下面写 数据分析代码`
- **WebAPI**（主控进程）：托管 gRPC Server、MQTT Client、系统状态服务、WaveformPublishService
- **数据通道**：`UISharedBuffer`（MMF 双通道波形） + gRPC 双向流（控制指令）
- **配置**：`DeviceConfig.json`（采集卡）、`RadarConfig.json`（激光器：仅功率/频率/串口）

### 缺失的链路
1. 缺少电压→物理量（Vis、Cn2）的全部反演算法
2. **缺少数据检测与工况判别模块**（架构文档 3.6 节）：基于环形缓冲区读写规则，实时判读信号质量、工况异常并推送告警
3. 缺少 MQTT 告警专属 Topic

### 2.2 核心算法模块（ConsoleApp1/Service/Lidar/）

所有算法类均放置于 `ConsoleApp1/Service/Lidar/` 目录下，由 `Analysis` 线程按流水线顺序调用：

```
电压数据 → [1] → [2] → [3] → [4] → [5] → Vis/Cn2 输出
```

#### 2.2.1 电压非线性修正 `VoltageNonlinearCorrector`
- 暗电流扣除：`V_corr = V_raw - V_dark(slice_index)`
- 分段增益校准：根据距离段索引查找标定系数 (k, b)，执行 `V_cal = k × V_corr + b`
- 非线性补偿：对高压区引入二次项修正
- 输出：修正后的电压数组

#### 2.2.2 光子计数反演 `PhotonCountInverter`
- 电压 → 光电流：`I = V_cal / R_load`
- 光电流 → 光功率：`P = I / Responsivity`
- 光功率 → 光子数：`N_photons = P × Δt / (h × c / λ)`（Δt 为采样间隔）
- 输出：光子计数剖面数组

#### 2.2.3 消光系数与后向散射联合反演 `ExtinctionBackscatterInverter`
- 距离校正信号：`S_corr(r) = N_photons(r) × r²`
- 对数距离导数：`d(ln S_corr) / dr`
- 斜率判据：根据斜率正负区分洁净区/污染区
- Fernald/Klett 反演：联立求解消光系数 α(r) 与后向散射系数 β(r)
- 输出：`ExtinctionProfile`（距离-消光系数数组）、`BackscatterProfile`

#### 2.2.4 能见度计算 `VisibilityCalculator`
- Koschmieder 定律：`Vis = ln(1/ε) / α_ext`（ε=0.02 为对比度阈值）
- 气象因子订正：根据温压湿修正空气分子散射分量
- 波长修正：`α(λ) = α(λ₀) × (λ/λ₀)^(-q)`（q 为 Angstrom 指数）
- 输出：单值 Vis + 高度分层能见度剖面

#### 2.2.5 Cn2 剖面反演 `Cn2ProfileInverter`
- 温度湍流谱模型：Tatarski 谱参数化
- 结构函数拟合：`D_T(r) = C_T² × r^(2/3)`
- Cn2 换算：`Cn² = C_T² × (79×10⁻⁶ × P / T²)²`
- 高度剖面校正：根据距离门逐段计算
- 输出：`Cn2Profile` 数组（距离-Cn2 值）

### 2.3 数据质量控制模块（ConsoleApp1/Service/Lidar/）

#### 2.3.1 异常跳点抑制 `SpikeSuppressor`
- 滑动窗口标准差法：3σ 阈值剔除离群点
- 中值滤波替代被剔除点

#### 2.3.2 时间平滑 `TimeSmoother`
- 指数加权移动平均（EWMA）累积多帧
- 平滑系数可配置（α=0.3~0.7）

#### 2.3.3 高度分层修正 `AltitudeLayerCorrector`
- 距离门对齐（根据采样率与光速计算每层高度）
- 重叠因子修正（近场盲区补偿）
- 大气分层假设校验（对流层/平流层过渡）

### 2.4 🆕 数据检测与工况判别模块（ConsoleApp1/Service/Lidar/Detection/）

> **对应架构文档 3.6 节**：独立后台高频线程，基于环形缓冲区读写规则，实时读取结构化采样数据，完成业务层智能分析与状态判别，输出工况结果与异常告警。

```
Analysis输出(Vis/Cn2/CH1/CH2) → [信号质量检测] → [工况智能判别] → [告警发布]
```

#### 2.4.1 信号质量检测器 `SignalQualityDetector`

**CH1/CH2 通道互相关分析**：
- 计算两通道皮尔逊互相关系数，判定信号一致性
- 互相关系数 < 阈值 → 标记 `SignalInconsistent`

**信号遮挡识别**：
- 监测后向散射信号幅度随距离的衰减模式
- 近场盲区探测：前 N 个距离门信号幅度低于噪声底限 → 标记 `NearFieldObstruction`
- 远场信号衰减异常：信号幅度持续低于 `SignalMinAmplitude` 超 `ObstructionDuration_ms` → 标记 `FarFieldObstruction`

**信号噪声含量检测**：
- 计算高频段（远距离噪声区）信号方差
- 方差 > `NoiseFloorMultiplier × 底噪` → 标记 `ExcessiveNoise`
- 功率谱密度分析：高频噪声能量占比超过阈值 → 标记 `HighFrequencyNoise`

**波形畸变检测**：
- 当前帧与历史正常帧模板的互相关度
- 互相关度 < `DistortionWaveformCorrelationMin` → 标记 `WaveformDistortion`
- 脉冲展宽检测：半高全宽（FWHM）与基准值偏差超过阈值 → 标记 `PulseBroadening`
- 多峰检测：单脉冲内出现多个峰值 → 标记 `MultiPeakAnomaly`

输出：`SignalQualityResult`（各项检测标志位 + 置信度）

#### 2.4.2 工况智能判别器 `ConditionAssessor`

**能见度工况判定**（基于 `VisibilityCalculator` 输出的 Vis 值）：
- Vis < VisLow_km → 工况等级 `LowVisibility`（严重）
- VisLow_km ≤ Vis < VisHigh_km → 工况等级 `ModerateVisibility`（一般）
- Vis ≥ VisHigh_km → 工况等级 `GoodVisibility`（正常）

**折射率异常判定**（基于 `Cn2ProfileInverter` 输出的 Cn2 剖面）：
- 相邻距离门 Cn2 跳变 > Cn2SpikeThreshold → 标记 `Cn2Spike`
- 整层 Cn2 均值超过历史基线 N 倍 → 标记 `Cn2Elevated`

**数据连续性检测**：
- 当前帧时间戳与上一帧间隔 > DataGapTimeout_ms → 标记 `DataGap`（数据断连）
- CH1/CH2 任一通道全零或全饱和 → 标记 `ChannelDead` / `ChannelSaturated`

**综合工况等级判定**：
- 汇总所有检测结果，输出最终工况等级：`Normal` / `Warning` / `Critical`
- 生成人类可读的工况描述字符串

输出：`ConditionAssessmentResult`

#### 2.4.3 告警发布器 `AlarmPublisher`

**告警事件模型** `LidarAlarmEvent`：
- `AlarmId`（Guid，唯一标识）
- `AlarmType`（枚举：LowVisibility / Cn2Spike / SignalObstruction / ExcessiveNoise / WaveformDistortion / DataGap / ChannelFault）
- `Severity`（枚举：Info / Warning / Critical）
- `Timestamp`（DateTime，事件时间）
- `VisValue` / `Cn2Value` / `AffectedChannel` / `Description`（辅助字段）
- `IsActive`（bool，告警是否持续激活中）

**MQTT 告警推送**：
- 复用现有 `MqttEventPublisher`，新增告警发布方法 `PublishLidarAlarmAsync()`
- 推送至告警专属 Topic：`daq/{MachineId}/lidar/alarm`
- QoS 1（保障可靠交付）+ Retain 最近一条告警快照
- 告警恢复时发送 `IsActive=false` 的清除消息

**本地告警日志**：
- 将告警事件写入本地日志文件 `Logs/lidar_alarm_{yyyyMMdd}.log`
- 记录工况状态快照（含 Vis、Cn2、信号质量各项指标），用于故障回溯

#### 2.4.4 检测流水线编排器 `DetectionPipelineOrchestrator`

协调检测模块的执行顺序和异常处理：

```
输入: AnalysisOutput(Vis/Cn2/CH1/CH2) + LidarDetectionConfig
  → SignalQualityDetector（信号质量检测）
  → ConditionAssessor（工况智能判别）
  → AlarmPublisher（告警发布）
  → 输出 LidarDetectionReport（检测报告）
```

**线程运行模式**：
- 独立于 `Analysis` 线程运行，通过 Channel 或共享缓冲区接收 Analysis 输出
- 遵循架构文档 3.6 节设计：线程优先级高于 UI/持久化线程，低于采集线程
- 单帧处理延迟 < 1ms（轻量级阈值比较 + 数值运算）

### 2.6 算法编排器 `LidarPipelineOrchestrator`

协调各算法模块的执行顺序、依赖关系和异常处理：

```
输入: Voltage_block + LidarAlgorithmConfig
  → 逐距离门迭代处理（避免全量数组分配）
  → 调用各子算法模块
  → 质量控制检查
  → 输出 PhysicalQuantityResult
```

## 5. 性能约束

- 零额外 GC：算法内部使用 `Span<T>` + `stackalloc` 避免堆分配
- 单帧反演延迟 < 5ms（46080 点/通道，双通道）
- 🆕 单帧检测延迟 < 1ms（阈值比较 + 数值运算，架构文档 3.6 节要求）
- 复用现有 `ArrayPool` 租用模式，不改动上游 Channel 数据流
- 兼容 Native AOT 编译（无反射、无 dynamic）
