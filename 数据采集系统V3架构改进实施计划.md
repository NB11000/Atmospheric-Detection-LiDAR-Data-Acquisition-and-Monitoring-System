# 数据采集系统V3架构改进实施计划

## 概述

本文档基于对《数据采集系统架构改进方案(V3).md》的分析和当前项目代码的审查，制定具体的改进实施计划。目标是将现有系统逐步演进为符合V3架构的工业级数据采集系统。

## 当前项目状态评估

### 已实现的核心组件 ✅

1. **进程分离架构** 
   - `AvaloniaApplication1`：UI主进程（Avalonia桌面应用）
   - `ConsoleApp1`：数据采集Worker进程

2. **gRPC双向通信**
   - `GrpcServiceImpl`：服务端实现，管理客户端连接
   - `GrpcClient`：客户端实现，接收服务端指令并响应
   - 支持双向流通信，指令响应机制完善

3. **共享内存框架**
   - `UISharedBuffer`：UI数据共享缓冲区（降采样波形数据）
   - `RawRingBuffer`：原始采样数据环形缓冲区
   - 无锁设计，支持高频数据交换

4. **基本的数据流与控制流分离**
   - UI通过共享内存直接读取数据
   - 控制命令通过gRPC传递
   - 支持基本的采集控制（START_AD、STOP_AD等）

### 架构差距分析 ⚠️

| 方面 | V3架构要求 | 当前实现状态 | 差距等级 |
|------|-----------|-------------|----------|
| **控制流与数据流分离** | 控制流通过Server→Worker，数据流直接到UI | 基本实现，但部分控制逻辑可能耦合 | 中等 |
| **UI无状态原则** | UI是无状态客户端，所有状态由Server管理 | UI可能保存了一些本地状态 | 高 |
| **REST API层** | UI/Browser通过HTTP REST API与Server通信 | 缺乏REST API层，直接调用服务 | 高 |
| **Web支持** | 支持Web Browser作为前端 | 只有Avalonia桌面UI | 中等 |
| **异常恢复机制** | Worker崩溃自动重启，UI崩溃不影响采集 | 未实现自动恢复机制 | 高 |
| **配置管理** | 配置由Server统一管理 | 配置文件分散，缺乏集中管理 | 中等 |
| **日志服务** | 日志服务由Server提供 | 日志分散在各自进程 | 中等 |
| **进程监控** | Server监控Worker健康状态 | 缺乏健康检查和自动重启 | 高 |

## 改进计划详细方案

### 第一阶段：架构完善（预计：1-2周）

#### 1.1 强化控制流与数据流分离
- **目标**：确保所有控制命令都通过ASP.NET Core Server的REST API
- **具体任务**：
  1. 梳理现有控制命令，识别直接调用点
  2. 创建REST API控制器，提供完整的控制接口
  3. 修改UI代码，通过HTTP API间接控制而非直接gRPC调用
  4. 完善gRPC命令集，支持完整的设备生命周期管理

#### 1.2 实现UI无状态化
- **目标**：将UI中的状态管理迁移到Server，UI仅作为视图层
- **具体任务**：
  1. 识别UI中保存的状态（设备句柄、采集状态、配置等）
  2. 创建状态管理服务，统一在Server端管理
  3. UI通过API获取状态，通过共享内存读取数据
  4. 确保UI重启后能自动恢复显示状态

#### 1.3 完善异常恢复机制
- **目标**：实现Worker进程监控与自动重启，确保系统健壮性
- **具体任务**：
  1. 实现Worker进程健康检查（心跳机制）
  2. 添加进程监控服务，自动检测并重启崩溃的Worker
  3. 确保UI崩溃不影响数据采集进程
  4. 实现优雅降级和故障转移

### 第二阶段：API与配置管理（预计：1周）

#### 2.1 实现REST API层
- **目标**：提供完整的HTTP REST API，支持UI和Web客户端
- **具体任务**：
  1. 设计REST API规范（OpenAPI/Swagger）
  2. 实现核心API控制器：
     - `AcquisitionController`：采集控制
     - `DeviceController`：设备管理
     - `ConfigController`：配置管理
     - `StatusController`：系统状态查询
     - `LogController`：日志查询
  3. 添加API身份验证和授权
  4. 提供API文档和测试工具

#### 2.2 统一配置管理
- **目标**：实现配置的集中管理和热更新
- **具体任务**：
  1. 创建`ConfigService`统一管理所有配置
  2. 支持配置文件热重载，Worker自动应用新配置
  3. 提供配置验证和默认值机制
  4. 实现配置版本管理和历史记录

### 第三阶段：扩展性与监控（预计：1-2周）

#### 3.1 添加Web支持
- **目标**：支持Web浏览器作为管理界面
- **具体任务**：
  1. 创建Web UI项目（建议使用Blazor或React）
  2. 复用现有REST API，实现Web管理界面
  3. 支持远程监控、配置和日志查看
  4. 实现响应式设计，支持多种设备

#### 3.2 完善日志服务
- **目标**：实现集中式日志收集和管理
- **具体任务**：
  1. 集成结构化日志框架（Serilog/ELK）
  2. 实现日志分级、过滤和搜索功能
  3. 添加操作审计日志，记录关键操作
  4. 提供日志导出和分析工具

#### 3.3 性能监控与优化
- **目标**：监控系统性能，优化关键路径
- **具体任务**：
  1. 添加性能指标收集（CPU、内存、数据吞吐量）
  2. 实现数据采集延迟监控和告警
  3. 优化共享内存缓冲区大小和性能
  4. 添加性能分析工具和报告

```csharp
public class WorkerManager : IWorkerManager, IDisposable
{
    private Process _workerProcess;
    private Timer _healthCheckTimer;
    private readonly ILogger<WorkerManager> _logger;
    private readonly WorkerConfig _config;
    
    public WorkerManager(ILogger<WorkerManager> logger, IOptions<WorkerConfig> config)
    {
        _logger = logger;
        _config = config.Value;
    }
    
    public void StartWorker()
    {
        try
        {
            // 构建启动参数
            var startInfo = new ProcessStartInfo
            {
                FileName = _config.WorkerExePath,
                Arguments = BuildWorkerArguments(),
                CreateNoWindow = _config.CreateNoWindow,
                UseShellExecute = false,
                RedirectStandardOutput = true,
                RedirectStandardError = true
            };
            
            _workerProcess = new Process { StartInfo = startInfo };
            _workerProcess.OutputDataReceived += OnWorkerOutput;
            _workerProcess.ErrorDataReceived += OnWorkerError;
            _workerProcess.Exited += OnWorkerExited;
            
            _workerProcess.Start();
            _workerProcess.BeginOutputReadLine();
            _workerProcess.BeginErrorReadLine();
            
            _logger.LogInformation($"Worker进程已启动，PID: {_workerProcess.Id}");
            
            // 启动健康检查
            StartHealthCheck();
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "启动Worker进程失败");
            throw;
        }
    }
    
    public void RestartWorkerIfDead()
    {
        if (_workerProcess == null || _workerProcess.HasExited)
        {
            _logger.LogWarning("Worker进程已停止，正在重启...");
            StartWorker();
        }
    }
```

### 配置管理服务

```csharp
public class ConfigService : IConfigService
{
    private readonly ConcurrentDictionary<string, object> _configs;
    private readonly FileSystemWatcher _configWatcher;
    private readonly ILogger<ConfigService> _logger;
    
    public ConfigService(ILogger<ConfigService> logger)
    {
        _logger = logger;
        _configs = new ConcurrentDictionary<string, object>();
        
        // 监听配置文件变化
        _configWatcher = new FileSystemWatcher(GetConfigDirectory());
        _configWatcher.Filter = "*.json";
        _configWatcher.Changed += OnConfigFileChanged;
        _configWatcher.EnableRaisingEvents = true;
        
        // 加载初始配置
        LoadAllConfigs();
    }
    
    public T GetConfig<T>(string configName) where T : class, new()
    {
        if (_configs.TryGetValue(configName, out var config))
            return config as T;
        
        // 加载配置
        var newConfig = LoadConfig<T>(configName);
        _configs[configName] = newConfig;
        return newConfig;
    }
    
    public void UpdateConfig<T>(string configName, T config) where T : class
    {
        // 验证配置
        ValidateConfig(config);
        
        // 保存到文件
        SaveConfig(configName, config);
        
        // 更新内存中的配置
        _configs[configName] = config;
        
        // 通知配置变更
        OnConfigChanged?.Invoke(this, new ConfigChangedEventArgs(configName));
        
        _logger.LogInformation($"配置已更新: {configName}");
    }
    
    private void OnConfigFileChanged(object sender, FileSystemEventArgs e)
    {
        // 配置文件变化时重新加载
        _logger.LogInformation($"检测到配置文件变化: {e.Name}");
        LoadConfig(e.Name);
        
        // 通知Worker重新加载配置
        NotifyWorkersConfigChanged(e.Name);
    }
}
```
