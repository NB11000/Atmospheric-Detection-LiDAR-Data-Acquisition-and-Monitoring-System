## 一、背景（Background）

### 问题（Problem）

#### 1. 启动器缺少设备参数配置入口

配置启动器（AvaloniaApplication_ConfigLauncher）仅提供 MQTT 连接配置和采集卡/激光器启停控制。要修改 USB1602 采集卡参数（采样率、通道、量程等）、LiDAR 反演算法参数（增益系数、K 常数、孔径等）、激光器硬件参数（功率、频率、串口）或持久化数据目录，必须手动编辑 `DeviceConfig.json` 文本文件，无 GUI。

#### 2. 设备配置类定义分散

`CaptureCardConfig`、`RadarConfig`、`LidarAlgorithmConfig`、`PersistenceSettings` 四个配置 DTO 仅存在于 WebAPI 项目中，启动器无法引用。如需通过 HTTP API 传输这些对象，需要重复定义类型。

#### 3. LiDAR 算法和持久化配置缺少 HTTP 管理端点

采集卡和激光器配置已有 HTTP 读写端点（`api/collector/command/config/*`、`api/laser/config/*`），但 LiDAR 算法和持久化配置仅能通过 MQTT RPC 通道修改，不符合 CONTEXT.md §13 规定的"启动器仅通过 HTTP 与 WebAPI 通信"原则。

#### 4. 持久化目录为相对路径

`PersistenceSettings.DataDirectory` 默认值 `"data"` 为相对路径，实际写入位置依赖 WebAPI 进程的工作目录，可能因启动方式不同而变化。

#### 5. 激光器配置页缺少单位标注和下拉框

激光功率、调制频率标签无单位，串口号和波特率为纯文本框，参考已有的子进程设置界面应使用 ComboBox 下拉选择。

---

## 二、解决方案（Solution）

### 整体思路

在启动器新增第 4 个 Tab "设备参数"，通过 HTTP API 与 WebAPI 通信，对采集卡/激光雷达算法/激光器/持久化四组配置提供表单化读写。WebAPI 侧补齐缺失的 HTTP 端点。将配置 DTO 统一迁至 SharedModels 消除类型重复。

### 具体实施

#### 1. 配置 DTO 迁移至 SharedModels

将 `CaptureCardConfig`、`RadarConfig`、`LidarAlgorithmConfig`、`PersistenceSettings` 四个文件从 `WebAPI/Config/` 和 `WebAPI/Models/` 物理移至 `SharedModels/`，namespace 统一为 `SharedModels`。WebAPI 和 ConfigLauncher 均通过 `ProjectReference` 共享同一套类型定义。

#### 2. WebAPI 新增 HTTP 配置端点

- **PersistenceController**（`api/persistence`）：config/read、config/update、config/default，其中 default 端点返回 `Path.GetFullPath()` 解析后的绝对路径
- **LidarController**（`api/lidar`）：config/read、config/update、config/default
- **LaserController** 补充：config/default 端点

#### 3. LauncherHttpClient 扩展

新增 12 个方法（4 组 × 3 操作：Get/Update/GetDefault），封装 GET/POST 请求、JSON 序列化、失败时返回 null。

#### 4. DeviceConfigViewModel（541 行）

单个 ViewModel 管理四节配置表单，使用 CommunityToolkit.Mvvm 的 `[ObservableProperty]` 和 `[RelayCommand]` 模式：

- **子切段切换**：`SelectedSubTabIndex` 变化时取消进行中的 HTTP 调用，自动加载新节
- **保存/刷新/恢复默认**：三 Command 独立操作，按当前切段调用对应的 LauncherHttpClient 方法
- **枚举字段**：采集卡六个枚举项使用 `int` 绑定 ComboBox `SelectedIndex`
- **雷达下拉**：串口（COM1~COM6）和波特率（9600~115200）使用 Index 属性 + 静态数组映射
- **持久化校验**：`SavePersistence()` 使用 `Path.IsPathRooted()` 拒绝相对路径
- **防重复提交**：`IsBusy` 标志 + `CancellationTokenSource` 控制并发

#### 5. DeviceConfigView + MainWindow 集成

- `IntEqualsConverter`：将 int 值转换为 bool，控制四节表单的可见性
- 顶部水平子 Tab 按钮切换四节
- 采集卡节：设备编号 ComboBox(0~15)、采样频率右侧 kHz 单位标注、枚举字段全部 ComboBox
- 激光器节：功率标签 (mW)、频率标签 (Hz)、串口 ComboBox(COM1~6)、波特率 ComboBox(9600~115200)
- LiDAR 算法节：11 项参数带物理单位标签
- 持久化节：TextBox + "选择文件夹"按钮（OpenFolderDialog）+ Watermark `D:\Data`
- MainWindow TabControl 新增第 4 个 Tab "设备参数"，WebAPI 连接状态联动可见性

#### 6. CONTEXT.md 更新

新增"设备参数面板 (Device Parameter Panel)"术语，更新"配置启动器"定义。

## 五、架构影响

```
配置启动器 (ConfigLauncher)
├── Tab "配置"          ← MQTT 配置 (已有)
├── Tab "控制"          ← 采集卡/激光器启停 (已有)
├── Tab "日志"          ← 占位 (已有)
└── Tab "设备参数"      ← ★ 新增
    ├── [采集卡]         ← api/collector/command/config/*
    ├── [激光器]         ← api/laser/config/*
    ├── [激光雷达算法]   ← api/lidar/config/*  ★ 新增端点
    └── [持久化]         ← api/persistence/config/* ★ 新增端点
```

| 维度 | 变更前 | 变更后 |
|------|--------|--------|
| 配置 DTO 位置 | 仅在 WebAPI | SharedModels（双方共享） |
| 配置修改方式 | 手动编辑 JSON | 启动器 GUI + HTTP API |
| LiDAR/Persistence HTTP 端点 | 不存在 | 各 3 个端点 |
| 持久化路径 | 相对路径 `"data"` | 强制绝对路径 |
| 枚举/串口输入 | TextBox 手工输入 | ComboBox 下拉选择 |
