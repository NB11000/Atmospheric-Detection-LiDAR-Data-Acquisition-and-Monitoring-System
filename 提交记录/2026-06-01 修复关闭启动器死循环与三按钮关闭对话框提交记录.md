## 一、背景（Background）

配置启动器（AvaloniaApplication_ConfigLauncher）是主控进程 WebAPI 的可选 GUI 前端，用户通过它配置 MQTT 参数、拉起 WebAPI 进程并通过 HTTP 操控采集卡和激光器。根据 ADR-0003，主控进程可脱离启动器独立运行。

### 问题（Problem）

#### 1. 关闭启动器窗口陷入死循环，用户无法关闭

`MainWindow.OnClosing` 中点击窗口 X 时流程为：`e.Cancel = true` → 弹对话框 → 用户点"是" → `e.Cancel = false; Close()` → 再次触发 `OnClosing` → `e.Cancel = true` → 又弹对话框，无限递归。用户必须手动退出 WebAPI 进程才能关闭启动器。

#### 2. 关闭对话框功能不足，无法选择是否顺带关闭 WebAPI

原有对话框仅提示"关闭启动器不会停止 WebAPI，是否继续？"（是/否），没有"顺带关闭 WebAPI"选项。用户期望关闭启动器时能够一键关闭 WebAPI。

#### 3. WebAPI 运行检测依赖进程引用，遗漏外部启动场景

原有检测使用 `_processManager.IsProcessRunning`，仅当启动器自身调用 `Start()` 拉起 WebAPI 时才为 true。若用户在外部手动启动 WebAPI，启动器无法感知 WebAPI 在线。

#### 4. 退出按钮绕过关闭流程

`ConfigViewModel.Exit()` 直接调用 `Environment.Exit(0)`，不弹任何对话框，也不给用户选择 WebAPI 去留的机会。

---

## 二、解决方案（Solution）

### 整体思路

修复 `OnClosing` 死循环（`_confirmedClose` 标志位），将确认对话框从二选一升级为三按钮多状态对话框（关闭WebAPI / 只关启动器 / 取消），WebAPI 检测改为 HTTP 连通性判断，退出按钮走统一关闭流程。

### 具体实施

#### 1. ViewModel 层：新增关闭和检测方法（`MainWindowViewModel`）

**`ShutdownWebApiAsync()`** — 封装 WebAPI 优雅关闭流程：
- 调用 `LauncherHttpClient.ShutdownWebApi()` 发起 `POST /api/system/shutdown`
- 检查返回值：HTTP 不可达时立即返回 false
- 调用 `WebApiProcessManager.StopAsync(BaseUrl, 3)` 等待进程退出（3 秒超时）

**`IsWebApiHttpReachableAsync()`** — 用 1 秒超时 HTTP GET 检测 WebAPI 在线状态，替代原有 `IsWebApiProcessRunning`

新增 `internal` 构造函数支持测试注入 `IWebApiProcessManager` 和 `HttpClient`

#### 2. View 层：修复死循环 + 三按钮多状态对话框（`MainWindow.axaml.cs`）

**死循环修复**：`_confirmedClose` 标志位。`OnClosing` 入口检测到此标志直接放行，`Close()` 前设置此标志，打破无限递归。

**三按钮多状态对话框**：

| 状态 | 显示内容 | 按钮 |
|------|----------|------|
| 初始 | "WebAPI 正在运行。关闭启动器时是否同时关闭 WebAPI？" | 关闭WebAPI / 只关启动器 / 取消 |
| 进度 | "正在关闭 WebAPI，请稍候..." + Indeterminate ProgressBar | 无（隐藏） |
| 错误 | 红色文字 "WebAPI 未能在 3 秒内退出。" + 三按钮 | 关闭WebAPI / 只关启动器 / 取消 |
| 二次错误 | 红色文字 "请手动关闭 WebAPI 进程。" + 三按钮 | 同上（点击"关闭WebAPI"显示提示而非重试） |

#### 3. 退出按钮统一流程（`ConfigViewModel`）

`Exit()` 从 `Environment.Exit(0)` 改为调用 `ExitRequested?.Invoke()` 委托，`MainWindowViewModel` 将委托接线为 `MainWindow.Close()`，走统一 `OnClosing → 对话框` 流程。

#### 4. 5 个单元测试（`MainWindowViewModelTests.cs`）

使用 FakeHttpMessageHandler + FakeProcessManager 覆盖三场景：
- `ShutdownWebApiAsync` 成功 / HTTP 不可达 / 超时
- `IsWebApiHttpReachableAsync` 可达 / 不可达

## 五、架构影响

| 维度 | 变更前 | 变更后 |
|------|--------|--------|
| WebAPI 运行检测 | 进程引用 (`IsWebApiProcessRunning`) | HTTP 连通性 (`IsWebApiHttpReachableAsync`) |
| 关闭对话框 | 二选一确认（是/否） | 三按钮多状态（关闭WebAPI / 只关启动器 / 取消） |

| 主控进程独立性 | 保持：可脱离启动器运行 | 保持："只关启动器"选项不停止 WebAPI |
