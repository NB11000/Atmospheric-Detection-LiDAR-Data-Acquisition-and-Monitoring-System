## 一、背景（Background）

持久化服务 `PersistenceService` 将 CSV 数据文件的输出路径硬编码为 `Path.Combine("data", fileName)`，无法按部署环境灵活配置。生产环境需要将数据写入挂载盘或自定义目录时，只能修改代码重新编译。

### 问题（Problem）

#### 1. 数据输出路径硬编码

`PersistenceService.RunLoopAsync()` 中第 103 行硬编码 `"data"` 字符串，不支持通过配置文件或环境变量覆盖。部署到不同机器时必须改代码。

#### 2. 死代码：注入但从未使用 MqttSettings

构造函数注入了 `IOptionsMonitor<MqttSettings>` 并存入 `_mqttSettings` 字段，但整个类中从未引用该字段。持久化服务不依赖 MQTT（`RequiresMqttConnection => false`），注入 MQTT 配置属于误导。

---

## 二、解决方案（Solution）

### 整体思路

遵循现有 `MqttSettings` 的 ASP.NET Core Options 模式，新增 `PersistenceSettings` 配置类，通过 `appsettings.json` 的 `Persistence:DataDirectory` 节点绑定，默认值 `"data"` 保持向后兼容。

### 具体实施

#### 1. 新增 PersistenceSettings 配置类

```csharp
public class PersistenceSettings
{
    public string DataDirectory { get; set; } = "data";
}
```

遵循与 `MqttSettings` 一致的命名约定和文件位置（`WebAPI/Models/`）。

#### 2. DI 容器注册绑定

```csharp
builder.Services.Configure<PersistenceSettings>(builder.Configuration.GetSection("Persistence"));
```

#### 3. 替换 PersistenceService 中的硬编码路径

```csharp
// 变更前
var filePath = Path.Combine("data", fileName);

// 变更后
var dataDir = _settings.CurrentValue.DataDirectory;
var filePath = Path.Combine(dataDir, fileName);
```

同时将构造函数参数从 `IOptionsMonitor<MqttSettings>` 改为 `IOptionsMonitor<PersistenceSettings>`，移除死代码。

#### 4. appsettings.json 显式声明默认值

```json
"Persistence": {
    "DataDirectory": "data"
}
```

运维可直接发现并修改此配置项。
