## 技术决策

### 反演算法

1. **能见度计算**：采用 Fernald (Klett) 后向积分法的简化形式——对经过距离平方校正的信号取自然对数，在远半段做 ln(S) vs r 线性回归，斜率 = -2α，代入 Koschmieder 公式 Vis = 3.912/α。避免近场重叠函数和边界噪声干扰。

2. **Cn² 计算**：采用球面波闪烁方差法。维护 N=100 帧环形缓冲区 `_ch1History[][]` / `_ch2History[][]`，逐距离门计算窗口内 `σI² = Var(ln(V1/V2))`，代入 `Cn² = K × σI² × D^(7/3) × L^(-11/6)`。前 N-1 帧窗口未满时输出哨兵值 -1.0。

3. **预处理流水线**：
   - 暗电流估计：从帧尾部取 `DarkCurrentSampleCount` 个点求均值，阈值钳位 `< 1e-12` 归零（解决 IEEE 754 浮点累积误差经 r² 放大后污染 ln 域线性回归）
   - 近场盲区掩蔽：r < BlindZoneDistance 时电压置零
   - 距离平方校正：V × r²
   - 双通道增益均衡：CH2 × GainEqualizationCoefficient

### 哨兵值约定

- Vis < 0 表示反演失败（信噪比不足、有效点过少）
- Cn² = -1.0 表示数据无效（窗口未满 或 单通道模式），物理 Cn² 恒正，-1 天然区分
- 使用者需检查 `if (sample.Cn2 < 0) skip`

## 踩坑记录

### IEEE 754 浮点精度陷阱

暗电流估计 `mean(tail) ≈ 0.05` 但 `0.05 × 50 / 50 ≠ 0.05`（double 累积误差 ~2e-17）。扣除后残余值虽微小，但经 r² 放大至 ~2e-11，再取 ln 得 ~-24.6，严重污染线性回归，导致 Vis 从预期的 7824m 暴跌至 106m（误差 98.6%）。

**修复**：暗电流扣除后 `< 1e-12` 的残余值强制钳位为 0。

### xUnit 10.0.7 API 变更

`Microsoft.Extensions.Configuration.Json` v10.0.7 中 `SetBasePath` / `AddJsonFile` / `AddJsonStream` API 签名变更，改用 `JsonDocument.Parse` + `GetProperty` + `GetRawText` + `JsonSerializer.Deserialize` 组合完成绑定测试。
