## 一、背景（Background）

MQTT 配置（BrokerHost、BrokerPort、ClientID、TLS 证书等）硬编码在 `appsettings.json` 中，非专业用户部署时需要手动编辑 JSON 文件，门槛高且容易出错。同时，当 MQTT 远程控制前端不可用时（网络故障、EMQX 宕机），系统缺乏本地控制通道来启停采集卡和激光器。

### 问题（Problem）

#### 1. MQTT 配置门槛高

用户不了解 MQTT 协议参数（ClientID 唯一性要求、TLS 证书路径、CA 证书配置），直接编辑 `appsettings.json` 容易填错导致服务无法启动。

#### 2. ClientID 碰撞风险

代码中 ClientID 默认值为 `daq-srv-01`，多台机器部署时会碰撞，EMQX 会踢掉旧连接而非拒绝新连接，导致设备反复断连。

#### 3. 缺乏本地控制通道

系统仅通过 MQTT-RPC 接收远程指令，当 MQTT Broker 不可用时无法控制采集卡和激光器。

#### 4. WebAPI 进程管理缺失

无优雅关闭机制，`Process.Kill()` 可能导致文件写入不完整或资源泄漏。

## 二、解决方案（Solution）

### 整体思路

新增 Avalonia 跨平台桌面应用 `AvaloniaApplication_ConfigLauncher`，作为**配置启动器 + 本地控制面板**，双入口架构（GUI 启动器 / WebAPI 直接启动均可）。通过 `.mqtt_configured` 标记文件判断首次运行，配置保存到 `appsettings.json` 的 `Mqtt` 节点。本地控制面板通过 HTTP 与 WebAPI 通信，新增 `/api/system/shutdown` 端点支持优雅关闭。

### 具体实施

#### 1. SharedModels 共享类库

新建 `SharedModels` 项目，放置跨项目共享的 DTO：`MqttSettings`（从 WebAPI 移入）、`CommandResult`（命令执行结果）、`SystemStateDto`（系统统一状态快照）。WebAPI 和 ConfigLauncher 双方引用，避免循环依赖。

#### 2. ConfigManager 配置管理模块

纯逻辑类，构造函数接收 `baseDirectory`，负责读写 `appsettings.json` 的 `Mqtt` 和 `Launcher` 节点，保留其他节点（`Logging`、`AllowedHosts`）不受影响。支持 `HasExistingConfig()` 和 `MarkConfigured()` 标记文件机制。

#### 3. ConfigViewModel 配置界面

首次运行显示完整配置表单（6 个核心字段 + 5 个高级字段在 Expander 中），`MachineId` 默认填入 `Environment.MachineName` 避免 ClientID 碰撞。已有配置显示摘要（用户名脱敏），支持切换到编辑模式。保存后自动重启 WebAPI 使配置生效。

#### 4. ControlViewModel 本地控制面板

8 个命令按钮（打开/关闭采集卡、开始/停止采集、连接/断开激光器、开启/关闭激光），手动刷新系统状态，实时显示执行结果。所有命令通过 `LauncherHttpClient` 以 HTTP 方式发送到 WebAPI。

#### 5. WebApiProcessManager 进程管理

启动 WebAPI.exe 并轮询等待就绪（1s 间隔，30s 超时）。通过 `POST /api/system/shutdown` 优雅关闭，超时 10s 后 `Process.Kill()` 兜底。支持进程存活检测 `IsProcessRunning`。

#### 6. WebAPI 新增 Shutdown 端点

`POST /api/system/shutdown` 调用 `IHostApplicationLifetime.StopApplication()` 触发 ASP.NET Core 优雅关闭流程。

#### 7. 双启动入口

GUI 启动器：启动配置界面 → 保存配置 → 启动 WebAPI → 控制面板。
直接启动：WebAPI.exe 直接从 `appsettings.json` 读取 MQTT 配置（无需启动器）。

## 五、架构影响

| 维度 | 变更前 | 变更后 |
|------|--------|--------|
| 启动方式 | 仅 WebAPI.exe 直接启动 | 双入口：GUI 启动器 / WebAPI 直接启动 |
| MQTT 配置 | 硬编码 appsettings.json | GUI 表单配置，MachineId 默认主机名 |

| WebAPI 关闭 | 无优雅关闭 | POST /api/system/shutdown 端点 |
| 控制通道 | 仅 MQTT-RPC 远程 | MQTT-RPC + HTTP 本地控制面板双通道 |
| 项目依赖 | WebAPI 独立 | WebAPI / ConfigLauncher 均引用 SharedModels |
